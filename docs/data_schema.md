# 제주아이랑 데이터 스키마

이 문서는 앱에서 사용하는 장소 데이터와 즐겨찾기 데이터의 최종 구조를 정의합니다.

## 1. 장소 데이터

- 운영 파일: `data/jeju-irang.csv`
- 데모 파일: `data/demo_data.csv`
- 기본 키: `place_id`
- 파일 형식: UTF-8 CSV, 첫 행에 컬럼명 포함

두 CSV는 같은 스키마를 사용합니다. 컬럼이 누락되어도 앱은 해당 컬럼을 빈 값으로
보완해 계속 실행합니다. `place_id`가 없거나 비어 있으면 화면 요소의 중복을 막기 위한
임시 ID(`AUTO-000001` 형식)를 실행 중에 생성합니다.

| 컬럼 | 형식 | 필수값 | 설명 | 누락 시 동작 |
|---|---|---:|---|---|
| `place_id` | 문자열 | 예 | 장소 고유 ID, 즐겨찾기 연결 키 | 임시 ID 자동 생성 |
| `place_name` | 문자열 | 예 | 사용자에게 표시할 장소명 | `정보 없음`으로 표시 |
| `category` | 문자열 | 아니요 | 관광지, 문화시설 등 시설유형 | 시설유형 필터에서 제외 |
| `space_type` | 문자열 | 아니요 | `실내`, `실외`, `실내/실외` | 공간 필터에서 제외 |
| `city_name` | 문자열 | 아니요 | 제주시 또는 서귀포시 | 빈 값으로 표시 |
| `legal_dong_name` | 문자열 | 아니요 | 읍·면·동 | 빈 값으로 표시 |
| `region_group` | 문자열 | 아니요 | 앱의 권역 필터 그룹 | 지역 필터에서 제외 |
| `road_address` | 문자열 | 아니요 | 도로명 주소 | `정보 없음`으로 표시 |
| `latitude` | 숫자 | 아니요 | WGS84 위도 | 지도·거리 계산에서 제외 |
| `longitude` | 숫자 | 아니요 | WGS84 경도 | 지도·거리 계산에서 제외 |
| `phone` | 문자열 | 아니요 | 문의 전화번호 | `정보 없음`으로 표시 |
| `website_url` | 문자열 | 아니요 | 공식 홈페이지 URL | 링크 없음 안내 |
| `closed_days` | 문자열 | 아니요 | 휴무일 | `정보 없음`으로 표시 |
| `opening_hours` | 문자열 | 아니요 | 운영시간 | `정보 없음`으로 표시 |
| `parking` | 문자열 | 아니요 | 무료·유료·불가 등 주차정보 | 주차 필터에서 제외 |
| `has_admission_fee` | 불리언 | 아니요 | 입장료 존재 여부 | 알 수 없음 처리 |
| `admission_fee_detail` | 문자열 | 아니요 | 요금 상세 | `정보 없음`으로 표시 |
| `has_age_limit` | 불리언 | 아니요 | 연령제한 존재 여부 | 알 수 없음 처리 |
| `age_limit_detail` | 문자열 | 아니요 | 연령제한 상세 | `정보 없음`으로 표시 |
| `nursing_room` | 불리언 | 아니요 | 수유실 여부 | 알 수 없음 처리 |
| `stroller_rental` | 불리언 | 아니요 | 유모차 대여 여부 | 알 수 없음 처리 |
| `reservation_url` | 문자열 | 아니요 | 예약 페이지 URL | 링크 없음 안내 |
| `resident_discount` | 불리언 | 아니요 | 제주도민 할인 여부 | 알 수 없음 처리 |
| `diaper_changing_table` | 불리언 | 아니요 | 기저귀 교환대 여부 | 알 수 없음 처리 |
| `photo_url` | 문자열 | 아니요 | 대표 이미지 URL | 기본 이미지 영역 표시 |
| `description` | 문자열 | 아니요 | 장소 한 줄 소개 | 기본 소개 문구 표시 |
| `review_summary` | 문자열 | 아니요 | 방문 참고사항 | 항목 숨김 |

불리언 컬럼은 `TRUE`/`FALSE`, `1`/`0`, `yes`/`no`, `예`/`아니오`를
인식하며 그 밖의 값은 알 수 없음으로 처리합니다.

## 2. 즐겨찾기 데이터

- 운영 저장소: Google Drive의 비공개 Google Sheets
- 워크시트: `bookmarks`
- 로컬 개발 대체 파일: `data/bookmarks.csv`
- 기본 키: `bookmark_id`
- 장소 데이터 연결 키: `place_id`

| 컬럼 | 형식 | 설명 |
|---|---|---|
| `bookmark_id` | 문자열 | 즐겨찾기 행의 고유 ID |
| `nickname` | 문자열 | 즐겨찾기 소유자를 구분하는 닉네임 |
| `place_id` | 문자열 | `jeju-irang.csv`의 `place_id`를 참조 |
| `created_at` | 문자열(ISO 날짜시간) | 저장 시각 |
| `password_salt` | 문자열 | 비밀번호 해시에 사용하는 무작위 salt |
| `password_hash` | 문자열 | 원문 대신 저장하는 비밀번호 해시 |
| `memo` | 문자열 | 사용자가 작성한 장소 메모 |
| `custom_category` | 문자열 | 사용자가 지정한 즐겨찾기 카테고리 |

한 장소는 여러 사용자의 즐겨찾기에 저장될 수 있으므로 장소와 즐겨찾기는
`place_id`를 기준으로 1:N 관계입니다. 즐겨찾기 CSV 다운로드에는 내부 식별자,
salt, 비밀번호 해시를 포함하지 않습니다.

## 3. 추천 규칙 데이터

- 파일: `data/rules.csv`

| 컬럼 | 형식 | 설명 |
|---|---|---|
| `column` | 문자열 | 비교하거나 정렬할 장소 컬럼 |
| `match_type` | 문자열 | `exact`, `nonempty`, `tiebreak_asc`, `limit` |
| `weight` | 숫자 | 추천 점수 가중치 |
| `value` | 문자열 또는 숫자 | 규칙의 고정값이나 노출 개수 |
| `enabled` | 불리언 | 규칙 사용 여부 |

파일이 없거나 스키마가 잘못되면 앱에 내장된 기본 추천 규칙을 사용합니다.

## 4. 오류가 발생하지 않도록 처리하는 방식

| 상황 | 원인 | 앱 동작 | 해결 방법 |
|---|---|---|---|
| 장소 CSV가 없음 | 파일 삭제, 이름 또는 경로 변경 | 빈 장소 목록으로 계속 실행 | `data/jeju-irang.csv` 또는 데모 모드의 `data/demo_data.csv` 복원 |
| CSV를 읽을 수 없음 | 빈 파일, 잘못된 CSV 형식, UTF-8이 아닌 인코딩 | 빈 장소 목록으로 계속 실행 | UTF-8 CSV로 다시 저장 |
| 장소 컬럼 누락 | 헤더 삭제 또는 컬럼명 변경 | 누락 컬럼을 빈 기본값으로 추가 | 위 표와 동일한 컬럼명을 첫 행에 복원 |
| 추천 규칙 오류 | `rules.csv` 누락 또는 스키마 불일치 | 기본 추천 규칙 사용 | 추천 규칙 표의 컬럼 복원 |
