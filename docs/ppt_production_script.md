# 제주아이랑 PPT 제작 스크립트

## 1. 발표 설계

- 발표 목표: 제주아이랑이 해결하려는 문제, 구현 방식, 데이터의 신뢰성과 데모 완성도를 전달한다.
- 발표 대상: 교육 운영진, 심사자, 동료 수강생
- 핵심 메시지: **아이 동반 외출에 필요한 흩어진 정보를 한곳에서 검색·비교·저장하고 예약까지 연결했다.**
- 권장 분량: 38장
- 목표 발표시간: 약 19분, 질의나 시연 지연을 위한 여유 1분 확보
- 이야기 흐름: `문제 → 통계 → 페르소나 → 핵심 기능 → 개발 → 데이터 → 데모 → 해결 사례 → 한계와 확장`

## 2. 가독성 원칙

- 화면비: 16:9
- 표지 제목: 50pt 이상
- 슬라이드 제목: 36~42pt
- 핵심 숫자: 48~64pt
- 본문: 24~30pt
- 표 안의 글자: 22pt 이상
- 출처: 10~12pt까지 허용하되 슬라이드 맨 아래 한 줄만 사용
- 한 슬라이드에는 하나의 주장만 둔다.
- 본문은 최대 3줄 또는 3개 항목으로 제한한다.
- 긴 설명은 발표 대본으로 옮기고 화면에는 핵심어만 남긴다.
- 데이터 스키마는 한 장에 몰아넣지 않고 세 장으로 나눈다.
- 표보다 숫자, 아이콘, 실제 화면 캡처를 우선한다.
- 화면 캡처는 1440×900 이상으로 준비하고 브라우저 탭과 주소창은 잘라낸다.
- 닉네임, 비밀번호, API 키, Google Sheets 식별정보는 캡처에 포함하지 않는다.

## 3. 화면 캡처 준비 목록

`assets/presentation/` 폴더에 다음 이름으로 준비한다.

| 파일명 | 캡처 내용 |
|---|---|
| `01-search-filter.png` | 지역·실내외·시설유형·육아 편의조건 필터와 검색 결과 |
| `02-map-distance.png` | 지도 화면과 현재 위치 기준 거리 정렬 결과 |
| `03-place-detail.png` | 장소 상세정보, 편의시설, 홈페이지·예약 링크 |
| `04-bookmarks.png` | 즐겨찾기, 카테고리, 메모 화면 |

제출용 PPT에는 위 캡처를 넣는다. 실제 발표에서는 데모 슬라이드 30~32를 띄운 뒤 웹 앱으로 전환해 직접 시연하고, 시간이 부족하거나 네트워크 문제가 생기면 캡처 화면으로 설명한다.
