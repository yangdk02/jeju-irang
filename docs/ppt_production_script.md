# 제주아이랑 PPT 제작 스크립트

## 발표 설계

- 발표 목표: 제주아이랑이 해결하려는 문제, 핵심 기능, 개발·협업 방식, 데이터와 데모 완성도를 전달한다.
- 발표 대상: 교육 운영진, 심사자, 동료 수강생
- 핵심 메시지: **아이 동반 외출에 필요한 흩어진 정보를 한곳에서 검색·비교·저장하고 예약까지 연결했다.**
- 권장 분량: 39장
- 목표 발표시간: 약 19분 30초, 시연 지연을 고려해 30초 여유 확보
- 이야기 흐름: `기획 배경 → 핵심 기능 → 개발·협업 → 데이터 → 데모 → 문제 해결 → 한계·계획`

## 공통 화면 체계

모든 본문 슬라이드는 위에서 아래로 다음 네 영역을 유지한다.

1. `섹션 중제목` — 20~22pt, 예: `01 기획 배경`
2. `7단계 진행바` — 현재 섹션은 주황색, 지난 섹션은 연한 주황색, 남은 섹션은 연한 회색
3. `짧은 제목` — 36~42pt, 가능하면 12자 안팎
4. `한 줄 설명` — 24~28pt, 제목 아래에서 이 슬라이드의 결론을 한 문장으로 요약

진행바의 단계명은 다음과 같이 고정한다.

`기획 배경 | 핵심 기능 | 개발·협업 | 데이터 | 데모 | 문제 해결 | 한계·계획`

| 슬라이드 | 섹션 중제목 | 진행바 활성 구간 |
|---:|---|---|
| 1 | 표지 | 진행바 없음 |
| 2 | 발표 안내 | 전체 단계 미리보기 |
| 3~11 | 01 기획 배경 | 기획 배경 |
| 12~17 | 02 핵심 기능 | 핵심 기능 |
| 18~21 | 03 개발·협업 | 개발·협업 |
| 22~30 | 04 데이터 | 데이터 |
| 31~33 | 05 데모 | 데모 |
| 34~36 | 06 문제 해결 | 문제 해결 |
| 37~38 | 07 한계·계획 | 한계·계획 |
| 39 | 마무리 | 전체 단계 완료 |

## 가독성 원칙

- 화면비: 16:9
- 표지 제목: 50pt 이상
- 슬라이드 제목: 36~42pt
- 핵심 숫자: 48~64pt
- 본문: 24~30pt
- 표 안의 글자: 22pt 이상
- 출처: 10~12pt까지 허용하되 맨 아래 한 줄만 사용
- 한 슬라이드에는 하나의 주장만 둔다.
- 본문은 최대 3줄 또는 3개 항목으로 제한한다.
- 긴 제목은 쓰지 않고, 설명은 제목 아래 한 줄이나 발표 대본으로 옮긴다.
- 데이터 스키마는 세 장으로 나누고, 표보다 숫자·아이콘·화면 캡처를 우선한다.
- 화면 캡처는 가로 1200px 이상으로 준비하고 브라우저 탭과 주소창은 잘라낸다.
- 닉네임, 비밀번호, API 키, 이메일, Google Sheets 식별정보는 캡처에 포함하지 않는다.
